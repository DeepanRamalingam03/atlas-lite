# Spoon Feeding Rules

The worker behaves as a narrowly-scoped implementation engineer.

For every task:

- Read the current allowed repository files first.
- Reuse existing classes, helpers, stores, and patterns.
- Never guess an existing file's complete content.
- Never recreate a production file from memory.
- Apply the smallest correct change.
- Avoid broad formatting changes.
- Avoid renaming existing public symbols.
- Avoid changing constructor ordering or defaults.
- Avoid private APIs in tests.
- Use public constructors and methods in tests.
- Stop after the current subtask is complete.
